# 验证场景

## 目标

确认这个 skill 在以下三类高频任务上可用：

1. 恢复已有知识库
2. 处理纯链接输入
3. 处理本地文件输入

## 场景 1：恢复场景

### 输入

- 已有 Obsidian 知识库根目录
- 关键文件：
  - `AGENTS.md`
  - `30_Indexes/知识库规则总览.md`

### 预期行为

- 正确识别最小恢复集
- 正确识别当前目录结构
- 正确识别当前知识分类
- 默认后续动作应为：
  - 先写 `00_Inbox`
  - 再沉淀知识
  - 最后更新索引

## 场景 2：链接收件场景

### 输入

- 一条 GitHub 链接或网页链接

### 预期行为

- 写入当天 `00_Inbox`
- 形成或更新正式知识笔记
- 分类落点符合现有中文目录
- 更新：
  - `30_Indexes/高频主题索引.md`
  - `30_Indexes/别名与关键词映射.md`

## 场景 3：文件收件场景

### 输入

- 一份 PDF、截图或图片文件

### 预期行为

- 写入当天 `00_Inbox`
- 复制原文件到 `90_Attachments/年-月/`
- 形成正式知识笔记或资料笔记
- 与已有主题建立关联
- 更新必要索引

## 判定标准

- 三个场景全部满足预期，记为通过。
- 任一场景缺少关键动作，记为未通过。
